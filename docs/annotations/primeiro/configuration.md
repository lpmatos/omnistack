# Semana Omnistack - Primeiro dia

## Configuração do Ambiente

A primeira ferramenta que vamos instalar é o NodeJS. O NodeJS vai ser a ferramenta que iremos utilizar para desenvolver o backend: desde a conexão com o banco de dados até a parte de rotas. 

Curiosamente o NodeJS também é utilizado no React e no ReactJS. 

Se você nunca ouviu falar sobre o NodeJS, o NodeJS basicamente é o motor do chrome utilizado para interpretar JavaScript. O JavaScript mais visual é interpretado pelo Chrome e a partir dele conseguirmos realizar algumas coisas, porém ele só existe ali dentro através de um Engine chamada V8. Em 2008 um cara pensou: e se a gente desacoplar o V8 do Chrome e tornar sua execução por fora? Foi assim que o NodeJS surgiu.

Portanto o NodeJS basicamente é o motor do Chrome tirado dele, permitindo interpretar e executar comandos JavaScript fora do Chrome.

Geralmente temos duas versos do NodeJS:

* LTS:
    * Versão com menos riscos no desenvolvimento.
    * Suporte estendido.
* Current:
    * Última versão lançada contendo as atualizações mais recentes.
    * Não é a versão mais estável.

Sempre queremos ter mais liberdado durante a instação do NodeJS em nosso sistema, seja Windows ou Linux, deveremos utilizar um gerenciador de pacotes para sua instação. A partir de um gerenciador de pacotes conseguimos ter facilidade para por exemplo:

* Remover o NodeJS
* O controle da versão.
* Atualizar.

O gerenciador de pacotes é uma ferramenta para facilitar a instalação de outras ferramentas em nossa máquina.
